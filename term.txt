온라인 설문조사 서비스 개발

Google Form, Survey Monkey 사이트를 참고하여 유사한 기능을 하는 웹서비스를 만들어봅시다.



https://www.google.com/forms

https://ko.surveymonkey.com/



다음과 같은 기능을 최소한 포함 하여야 한다.



1. 사용자 관리

- 회원 가입

- 로그인

- 회원 목록 관리를 위한 관리자 페이지

- (Option) 회원 정보 수정, 비밀번호 수정, 회원 탈퇴

- (Option) Facebook 로그인

- (Option) 비밀번호 재설정 이메일 전송

- (Option) 비밀번호 암호화



2. 설문 디자인

- 설문 디자인은 로그인된 회원만 수행할 수 있다. 

- 설문의 제목, 전체 설명 입력

- 사용자는 다양한 질문 형태를 나열하여 설문지를 디자인 할 수 있음

- 설문 문항의 추가, 수정, 삭제가 가능하다.

- 질문 형태로서 객관식, 단일 입력(1줄 text), 의견(long text) 입력은 최소한 구현 되어야 함.

- (Option) 객관식의 경우 보기에 없을 경우 '기타'를 선택하고 단일 입력을 추가로 할 수 있는 옵션 제공

- (Option) 객관식일 경우 단일 선택, 복수 선택을 구별할 수 있는 옵션 제공

- (Option) 날짜 입력, 시간 입력, 1 - 5 점 점수 선택 Scale (Radio button) 등의 추가 질문 형태 제공

- (Option) 필수/선택 입력 옵션을 두어 특정 질문에는 사용자가 반드시 답을 입력해야 되도록 제공



3. 설문 수집

- 완성된 설문은 별도의 URL을 통하여 설문 조사를 수행할 수 있다.

- (Option) 설문 초대: 이메일 주소를 입력하여 설문자를 초대하는 기능. 초대 이메일이 발송되어야 함

- (Option) 설문 기간: 특정 기간을 설정하여 그 기간 동안만 설문 조사 가능

- (Option) 중복 설문 방지: 이메일 주소 등을 입력받아 동일한 사람이 2번 설문에 응답하지 못하도록



4. 설문 결과 Visualize

- 설문자는 각 설문별로 응답자 현황을 확인할 수 있다.

- 각 질문별로 사용자의 응답을 확인할 수 있다.

- 주요 질문 형태별로 그에 맞는 요약 정보를 제공한다. (예: 객관식의 경우 각 문항별 선택자 수, Scale의 경우 평균 점수)

- (Option) 요약 정보를 적절한 형태의 chart형태로 제공한다. 예: 파이 그래프, 막대 그래프 등. 단 이때 chart는 순수 JavaScript 기술을 이용한 것만 활용하며 Flash 등을 이용한 chart는 활용 금지.

- (Option) 설문 결과 Excel, CSV로 내려받기



5. 최종보고서

- 최종 보고서에는 왜, 어떻게 이 프로그램을 제작하였는지에 대한 이유를 기술한다. 불필요하게 전체 소스를 dump하지 않는다. 

- Google Form, Survey Monkey 등의 관련 서비스를 조사하고, 각 서비스의 특징을 본인이 구현한 서비스와 비교 설명한다.

- 본인의 프로그램이 제공하는 기능, 이를 위해서 본인이 설계한 내용, 데이터 디자인, 사이트 디자인, SW 구조 등을 설명한다.

- 최종 보고서 첫 페이지에 반드시 접속 가능한 URL을 제시하여 확인할 수 있도록 한다.

- (Option) ER-Diagram, Class Diagram, System Architecture 등을 제시한다. 

- (Option) Github을 사용하는 경우 Github URL을 첫 페이지에 명시하기 바람.



기타

- 모든 Modern web browser에서 정상적으로 수행 가능해야 한다. (최신 버전의 IE, Safari, Chrome, Firefox)

- 본 수업에서 배운 기술을 활용한다. (NodeJS, MongoDB 필수 사용)

- 본 과제의 결과물은 실제 웹 상에서 확인 가능해야 한다. Heroku와 Mongolab을 활용하면 추가 비용없이 가능함

- (Option) 모바일 Browser에서도 사용가능한 Responsive 디자인을 사용한다.

- (Option) Ajax 기술을 활용하여 설문 디자인 페이지를 제작한다.

- (Option) Github을 사용하여 전체 소스코드를 관리하고, 공개한다. 전체 프로젝트 진행 기간에 걸쳐 지속적으로 사용해야 하며, 이 경우 마지막에 한 두번의 commit만 존재하는 경우 github을 활용하지 않은 것으로 간주
